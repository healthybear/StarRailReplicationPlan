# 《星穹铁道剧情复现计划》项目进度表

本文档以**里程碑 + 相对周次（W1/W2…）**组织项目全流程，适用于当前阶段（已完成需求与设计文档产出）到后续 UI 设计、开发、测试、验收、发布与迭代。

- **说明**：周次为相对时间，默认 1 周=5 个工作日；可在确定团队规模/目标上线日期后，将周次映射为实际日期。
- **角色**：PM=项目经理（统筹/排期/风险/验收），TL=技术经理/技术负责人（架构/质量/技术决策），UX=UI/UX，DEV=研发，QA=测试，OPS=运维/发布（如有）。

## 五、关键风险与缓冲（建议）

- **技术选型未定导致返工**：将 TS-1/2/3 前置到 W1–W3，并在 WBS/进度表中绑定决策点（冻结后再大规模开发）。\n- **视野隔离/信息不对称难实现**：在 Phase 1 早期（W2）就实现 VM 过滤接口，并以 UI-02 验收用例持续回归。\n- **锚点标注工作量**：Phase 1 只做单节点锚点与对比；Phase 2 再扩充锚点规模与工具化。\n- **多 Agent 冲突复杂**：明确 Phase 2 作为主攻点，Phase 1 仅保证单/双角色可运行。\n- **缓冲建议**：每个 Phase 的测试与发布预留 1 周缓冲（W5、W10、W14）。\n+
