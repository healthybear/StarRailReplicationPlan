## 🚀 快速启动

### 环境要求

- Node.js >= 20.0.0
- pnpm >= 8.0.0
- Deepseek API Key 或 Claude API Key

### 启动步骤

\`\`\`bash

# 1. 安装依赖

pnpm install

# 2. 配置环境变量

cp .env.example .env

# 编辑 .env，填入 DEEPSEEK_API_KEY

# 3. 构建项目

pnpm build

# 4. 启动后端（终端 1）

cd packages/api
pnpm start:dev

# 5. 启动前端（终端 2）

cd packages/web
pnpm dev

# 6. 访问应用

# Web UI: http://localhost:5173

# API 文档: http://localhost:3000/api/docs

\`\`\`

## ⚠️ 已知限制

### 数据持久化

- Character 和 Scene 模块使用内存存储
- 服务器重启后数据丢失
- 需要集成 Core 包的文件存储

### 认证授权

- Auth 模块仅提供基础框架
- 未实现真实的用户认证
- 所有端点未添加权限控制

### LLM 集成

- Story Advancement 在测试环境中可能失败
- 需要配置 LLM API Key 和网络环境

## 🔧 常见问题

### Q1: 如何配置 LLM API？

编辑 `.env` 文件：
\`\`\`bash
DEEPSEEK_API_KEY=sk-xxxxx
\`\`\`

编辑 `config/llm.yaml`：
\`\`\`yaml
defaultProvider: deepseek
providers:
deepseek:
enabled: true
model: deepseek-chat
\`\`\`

### Q2: 如何添加新的人物？

**方式 1: Web UI**

1. 访问 http://localhost:5173
2. 点击 "人物管理"
3. 点击 "创建人物"
4. 填写表单并提交

**方式 2: API**
\`\`\`bash
curl -X POST http://localhost:3000/api/characters \
 -H "Content-Type: application/json" \
 -d '{"characterId":"xxx","name":"xxx",...}'
\`\`\`

**方式 3: 配置文件**
在 `config/characters/` 目录下创建 YAML 文件

### Q3: 如何查看 API 文档？

启动 API 服务器后访问：
http://localhost:3000/api/docs
