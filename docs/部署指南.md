# 部署指南

本文档介绍如何将《星穹铁道剧情复现计划》部署到生产环境。

## 📋 部署前准备

### 服务器要求

- **操作系统**: Linux (Ubuntu 20.04+ 推荐) 或 Windows Server
- **CPU**: 2 核心以上
- **内存**: 4GB 以上
- **存储**: 20GB 以上
- **Node.js**: 20.0.0 以上
- **pnpm**: 8.0.0 以上

### 域名和 SSL 证书（可选）

如果需要通过域名访问，准备：

- 域名（如 `star-rail.example.com`）
- SSL 证书（推荐使用 Let's Encrypt）

## 🚀 部署步骤

### 1. 克隆项目

\`\`\`bash
git clone <repository-url>
cd StarRailReplicationPlan
\`\`\`

### 2. 安装依赖

\`\`\`bash
pnpm install --prod
\`\`\`

### 3. 配置环境变量

\`\`\`bash
cp .env.example .env
nano .env
\`\`\`

生产环境配置示例：

\`\`\`bash

# LLM API Keys

DEEPSEEK_API_KEY=sk-xxxxx
ANTHROPIC_API_KEY=sk-ant-xxxxx

# 应用配置

NODE_ENV=production
LOG_LEVEL=warn
PORT=3000

# 数据库（如果使用）

# DATABASE_URL=postgresql://user:pass@localhost:5432/starr ail

\`\`\`

### 4. 构建项目

\`\`\`bash
pnpm build
\`\`\`

### 5. 启动服务

#### 方式 1: 使用 PM2（推荐）

\`\`\`bash

# 安装 PM2

npm install -g pm2

# 启动后端 API

cd packages/api
pm2 start dist/main.js --name star-rail-api

# 启动前端（如果需要）

cd ../web
pm2 start "pnpm preview" --name star-rail-web

# 保存 PM2 配置

pm2 save
pm2 startup
\`\`\`

#### 方式 2: 使用 systemd

创建服务文件 `/etc/systemd/system/star-rail-api.service`:

\`\`\`ini
[Unit]
Description=Star Rail API Server
After=network.target

[Service]
Type=simple
User=your-user
WorkingDirectory=/path/to/StarRailReplicationPlan/packages/api
ExecStart=/usr/bin/node dist/main.js
Restart=on-failure
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
\`\`\`

启动服务：

\`\`\`bash
sudo systemctl daemon-reload
sudo systemctl enable star-rail-api
sudo systemctl start star-rail-api
sudo systemctl status star-rail-api
\`\`\`

### 6. 配置 Nginx 反向代理（可选）

创建 Nginx 配置文件 `/etc/nginx/sites-available/star-rail`:

\`\`\`nginx
server {
listen 80;
server_name star-rail.example.com;

    # API 代理
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 前端静态文件
    location / {
        root /path/to/StarRailReplicationPlan/packages/web/dist;
        try_files $uri $uri/ /index.html;
    }

}
\`\`\`

启用配置：

\`\`\`bash
sudo ln -s /etc/nginx/sites-available/star-rail /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
\`\`\`

### 7. 配置 SSL（可选）

使用 Certbot 获取 Let's Encrypt 证书：

\`\`\`bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d star-rail.example.com
\`\`\`

## 🔒 安全配置

### 1. 防火墙配置

\`\`\`bash

# 允许 HTTP 和 HTTPS

sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 允许 SSH

sudo ufw allow 22/tcp

# 启用防火墙

sudo ufw enable
\`\`\`

### 2. 环境变量保护

确保 `.env` 文件权限正确：

\`\`\`bash
chmod 600 .env
\`\`\`

### 3. 日志管理

配置日志轮转 `/etc/logrotate.d/star-rail`:

\`\`\`
/path/to/StarRailReplicationPlan/logs/\*.log {
daily
rotate 7
compress
delaycompress
notifempty
create 0640 your-user your-group
}
\`\`\`

## 🐳 Docker 部署（可选）

### 构建镜像

\`\`\`bash
docker build -t star-rail-api -f packages/api/Dockerfile .
docker build -t star-rail-web -f packages/web/Dockerfile .
\`\`\`

### 运行容器

\`\`\`bash

# 后端 API

docker run -d \
 --name star-rail-api \
 -p 3000:3000 \
 -v $(pwd)/data:/app/data \
 -v $(pwd)/.env:/app/.env \
 star-rail-api

# 前端

docker run -d \
 --name star-rail-web \
 -p 5173:5173 \
 star-rail-web
\`\`\`

### 使用 Docker Compose

创建 `docker-compose.yml`:

\`\`\`yaml
version: '3.8'

services:
api:
build:
context: .
dockerfile: packages/api/Dockerfile
ports: - "3000:3000"
volumes: - ./data:/app/data - ./.env:/app/.env
restart: unless-stopped

web:
build:
context: .
dockerfile: packages/web/Dockerfile
ports: - "5173:5173"
depends_on: - api
restart: unless-stopped
\`\`\`

启动：

\`\`\`bash
docker-compose up -d
\`\`\`
