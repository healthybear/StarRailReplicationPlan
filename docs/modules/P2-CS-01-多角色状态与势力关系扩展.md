# P2-CS-01 多角色状态与势力/关系扩展

## 一、模块概述

- **模块目标**: 扩展人物关系系统，支持多维度关系网和势力体系
- **对应 WBS 任务 ID**: P2-CS-01
- **依赖模块**: P1-CS-01, P1-CS-02, P1-CS-03
- **被依赖模块**: P2-CA-01, P2-SO-01, P2-AE-01

## 二、功能清单

### 2.1 势力系统（FactionService）

| 功能名称      | 功能描述                       | 对应的类/方法                                  | 文件位置                                               |
| ------------- | ------------------------------ | ---------------------------------------------- | ------------------------------------------------------ |
| 势力注册      | 注册和管理势力定义             | `FactionService.registerFaction()`             | `packages/core/src/character-state/faction.service.ts` |

| 盟友/敌对查询 | 查询势力的盟友和敌对势力       | `FactionService.getAllies()`, `getEnemies()`   | `packages/core/src/character-state/faction.service.ts` |

## 三、API 接口

### 3.1 FactionService

```typescript
class FactionService {
  // 注册势力
  registerFaction(faction: Faction): void;

  // 加载势力配置
  loadFactionConfig(config: FactionConfig): void;

  // 获取势力
  getFaction(factionId: string): Faction | undefined;

  // 成员管理
  addMember(factionId: string, characterId: string): boolean;
  removeMember(factionId: string, characterId: string): boolean;
  getCharacterFaction(characterId: string): Faction | undefined;

  // 势力关系
  getFactionRelationship(
    factionId: string,
    targetFactionId: string
  ): FactionRelationship;
  updateFactionRelationship(
    factionId: string,
    targetFactionId: string,
    updates: Partial<FactionRelationship>
  ): boolean;
  areAllies(factionId1: string, factionId2: string): boolean;
  areHostile(factionId1: string, factionId2: string): boolean;
  getAllies(factionId: string): Faction[];
  getEnemies(factionId: string): Faction[];
}
```

### 3.2 RelationshipManager

```typescript
class RelationshipManager {
  // 获取关系
  getRelationship(character: Character, targetId: string): Relationship;

  // 更新关系
  updateRelationshipDimension(
    character: Character,
    targetId: string,
    dimension: keyof Relationship,
    value: number,
    reason?: string
  ): void;
  updateRelationship(
    character: Character,
    targetId: string,
    updates: Partial<Relationship>,
    reason?: string
  ): void;

  // 关系标签
  addRelationshipTag(character: Character, targetId: string, tag: string): void;
  removeRelationshipTag(
    character: Character,
    targetId: string,
    tag: string
  ): void;
  hasRelationshipTag(
    character: Character,
    targetId: string,
    tag: string
  ): boolean;

  // 关系摘要
  getRelationshipSummary(
    character: Character,
    targetId: string
  ): {
    overall: 'positive' | 'neutral' | 'negative';
    dominant: string;
    tags: string[];
  };

  // 变更历史
  getChangeHistory(
    characterId?: string,
    limit?: number
  ): RelationshipChangeRecord[];
}
```

## 四、配置文件

### 4.1 势力配置

**文件路径**: `config/factions/*.yaml`

**配置示例**:

```yaml
id: faction_stellaron_hunters
name: 星核猎手
description: 追寻星核的神秘组织
ideology: 追寻命运的终结
resources: 85
influence: 70

relationships:
  faction_astral_express:
    attitude: -0.2
    alliance: false

members:
  - character_kafka
  - character_blade

leader: character_kafka
```

### 4.2 势力事件触发表

**文件路径**: `config/triggers/faction_events.yaml`

**配置示例**:

```yaml
version: '2.0'
rules:
  - id: faction_conflict_trust_loss
    name: 势力冲突导致信任度下降
    eventType: faction_conflict
    effects:
      - target: relationship.trust
        changeType: delta
        value: -0.15
```

## 七、已知限制

1. **势力层级**: 当前不支持势力的层级结构（如子势力、分支）
2. **势力资源**: 势力资源和影响力目前仅为数值，未实现具体的资源管理逻辑
3. **关系推理**: 关系变化暂不支持自动推理（如 A 信任 B，B 信任 C，则 A 可能信任 C）
4. **历史查询**: 变更历史暂不支持复杂查询（如按时间范围、按事件类型）
